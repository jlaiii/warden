Features

Everything you need to inspect your system

Warden combines a 1:1 Windows Task Manager clone with a full anti-rootkit security suite — all in a single .py file.

🔍

Cross-View Detection

Compares user-mode API results against raw NT kernel syscalls. If a process shows up via NtQuerySystemInformation but not psutil, it's hidden.

🧵

12 Scanner Categories

Hidden processes, threads, modules/DLLs, services, files, MBR bootkits, ADS, registry keys, inline hooks — plus SSDT/IDT/IRP awareness.

📊

Full Task Manager GUI

8 tabs including Processes (with smart grouping), Performance graphs, App History, Details, Services, Startup, Users, and Security.

🛡️

Anti-Detection System

Randomized mutex names, temp-copy self-protection with different file hashes, XOR+base64 string obfuscation — malware can't block what it can't find.

⚡

Dual Mode

Full GUI via tkinter or CLI mode with text progress bars and JSON output. Works as standalone scanner or integrated tab.

🔬

Inline Hook Detection

Compares on-disk .text sections of critical DLLs against their in-memory code via ReadProcessMemory. Detects API hooks in ntdll, kernel32, and more.

Detection Engine

12 security scanners, one click

Each scanner uses two different enumeration paths. Any discrepancy between them signals a potential rootkit hiding something.

Scanner Severity Admin Detection Method
Hidden Processes Critical Yes psutil.pids() vs NtQuerySystemInformation
Hidden Threads Critical Yes CreateToolhelp32Snapshot vs NT syscall
Hidden Modules / DLLs High Yes Module32First vs SystemModuleInformation
Hidden Services High No SCM vs raw HKLM registry
Hidden Files High No FindFirstFile vs NtQueryDirectoryFile
Hidden Disk Sectors High Yes MBR boot signature check via \\.\PhysicalDrive0
Alternate Data Streams Medium No FindFirstStreamW on system directories
Hidden Registry Keys High No RegEnumKey vs NtEnumerateKey
Inline Hooks Critical No On-disk .text vs in-memory ReadProcessMemory
SSDT Hooks Info — Requires kernel driver (informational)
IDT Hooks Info — Requires kernel driver (informational)
IRP Hooks Info — Requires kernel driver (informational)
How It Works

Cross-view detection explained

Rootkits hide by hooking user-mode API functions. Warden calls the same function through two different paths — if one path sees something the other doesn't, it's flagged.

Get Started

Installation

Zero external dependencies beyond psutil. Runs on any Windows 10/11 machine with Python 3.10+.

# Clone the repository $ git clone https://github.com/jlaiii/warden.git $ cd warden # Install the only dependency $ pip install psutil # Launch the Task Manager with Security tab $ python task_manager.py # Or run the standalone scanner — quick scan $ python warden_scanner.py --cli --quick # Full deep scan with JSON output $ python warden_scanner.py --json -o report.json

⚠️ Run as Administrator for full detection capability — NT-level process scanning, MBR checks, and kernel module enumeration all require elevated privileges.

Smart Process Grouping

Same-name processes auto-collapse into groups with aggregated CPU, memory, threads, and handles — just like the real Task Manager. Groups persist expanded state across refreshes.

📈

Live Performance Graphs

Per-core CPU bars, memory composition, disk I/O, network throughput, and GPU monitoring via nvidia-smi. Scrollable dashboard with real-time updates.

⚙️

Service Control

Start, stop, and restart Windows services. Color-coded by status. Search and filter 270+ services. Admin-elevated controls.

🔧

Process Actions

End task, end process tree, set priority class (Realtime through Low), set CPU affinity, create dump files via MiniDumpWriteDump, analyze wait chains, open file location, search online.

Architecture

Clean, modular design

Four Python files, each with a clear responsibility. No framework, no bloat.

🖥️

task_manager.py

Main application — 8-tab GUI built with tkinter/ttk. Includes the complete Task Manager clone and integrated Security tab. 3,600+ lines.

🔬

sec_scanner_engine.py

Core detection engine — all 12 anti-rootkit scanners, NT API wrappers via ctypes, cross-view data model, and scan orchestrator. 1,450+ lines.

🛡️

anti_detect.py

Anti-detection system — randomized mutex, XOR+base64 string obfuscation, temp-copy self-protection with hash randomization, atexit cleanup.

🚀

warden_scanner.py

Standalone interface — CLI mode with progress bars and text/JSON output, plus a self-contained tkinter GUI mode. Supports quick and full scan presets.
